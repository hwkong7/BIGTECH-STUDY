## 데이터베이스 관리 시스템(DBMS)

> 데이터베이스를 관리하기 위한 프로그램
> 
- DBMS는 사용자와 직접적으로 상호작용하기보다는 사용자(개발자)가 만든 프로그램과 상호작용하며 실행됨.
- 다른 컴퓨터의 응용 프로그램에서도 DBMS에 접속 가능

![image.png](attachment:ea4422d0-6d24-44fc-8864-3594ef0cae46:image.png)

**DBMS의 종류**

- 관계형 데이터베이스 관리 시스템(RDBMS)
- NoSQL 데이터베이스 관리 시스템(NoSQL DBMS)

---

## 응용 프로그램이 DBMS를 이용하는 과정

- 주로 쓰리 웨이 핸드셰이크를 통한 TCP 연결을 맺음.
    - 때로는 데이터베이스에 접속하기 위한 인증이 필요하기도 함.
- DBMS 클라이언트는 DBMS의 데이터베이스 언어(SQL)를 통해 데이터베이스를 다룰 수 있게 됨.
    - SQL: RDBMS에서 데이터를 조작하고 관리하기 위한 데이터베이스 언어

---

## SQL의 종류

- DDL(Data Definition Language)
- DML(Data Manipulation Language)
- DCL(Data Control Language)
- TCL(Transaction Control Language)

---

## 파일 시스템 대신 데이터베이스를 사용하는 이유

- 데이터 일관성 및 무결성 제공이 어려움.
    - 보통 여러 명의 사용자, 프로그램이 동시다발적으로 데이터베이스를 이용 → 레이스 컨디션 문제 발생 → 데이터의 일관성 훼손
- 불필요한 중복 저장이 많아짐.
- 데이터 변경 시 연관 데이터 변경이 어려움.
- 정교한 검색이 어려움.
    - 파일 내 문자열 검색에 국한
- 백업 및 복구가 어려움.
    - 데이터베이스는 백업과 복구 기능을 제공함.

## 데이터베이스의 저장 단위

- 엔티티: 독립적으로 존재할 수 있는 객체
    - 쉽게 말해, 어떠한 특성을 가진 대상
    - 데이터베이스에 저장 가능한 대상
    - ex. 김한빛, 노트북, 키보드 등
- 속성: 엔티티의 특성
    - ex. 구매자 ID, 구매자의 이름과 성별, 제품 번호와 이름 등
- 도메인: 엔티티의 속성이 가질 수 있는 값의 집합
    - ex. 구매자 성별의 도메인 ⇒ (남자, 여자)
- 엔티티 집합: 같은 속성을 공유하는 개별 엔티티

---

## DBMS가 엔티티 집합을 표현·저장하는 방식

- 레코드: 데이터베이스에 저장된 대상으로, 데이터베이스에 기록된 각각의 엔티티
- 필드: 데이터베이스에 저장된 엔티티 속성
    - 차수: 필드의 수
    - 카디날리티(cardinality): 한 필드에 대한 고유값의 수
    - 카디날리티가 낮을 수록 중복된 속성이 많음.

 ⇒ 다양한 **속성**을 지닌 **엔티티**들이 다양한 **필드**를 지닌 **레코드**로써 데이터베이스에 저장됨.

- 릴레이션: RDBMS에서 표현되는 이차원 테이블 형태의 엔티티 집합
    - 개별 레코드 → 테이블의 행
    - 필드 → 테이블의 열
    
    ![image.png](attachment:9afba278-49ec-48ab-9d31-465d5841100d:35a01282-f44b-4b60-a980-1e76e66877f4.png)
    
- 컬렉션: NoSQL DBMS에서 표현되는 엔티티 집합
    - 개별 레코드 → Json 형태의 데이터인 도큐먼트
    - 필드 → Json의 키 형태

## 스키마

> 데이터베이스에 저장되는 레코드의 구조와 제약 조건을 정의한 것
> 
- RDBMS와 NoSQL을 구분하는 주요 기준: 스키마의 유무
    - 스키마 O → 정형화된 데이터를 저장하고 관리하기 용이
    - 스미카 X → 유연한 형탱의 데이터를 저장하고 관리하기 용이
- 스키마에는 **테이블의 구조**와 **각 필드의 데이터 타입**, **제약 조건** 등이 정의됨.

## 트랜잭션

> 데이터베이스가 처리하는 작업의 단위
> 
- 초당 트랜잭션(TPS)로 데이터베이스의 작업 성능을 나타냄.
- 트랜잭션은 하나의 쿼리만 포함하는 것이 아닌, 2개 이상의 쿼리가 포함될 수 있음.
    - ex. “한빛”이가 “빛한”이에게 5,000원을 이체하는 트랜잭션에 포함되는 쿼리
        - 한빛이의 계좌 잔액을 5,000원 감소시키기
        - 빛한이의 계좌 잔액을 5,000원 증가시키기

---

## ACID

> 트랜잭션이 지켜야 하는 4가지 성질
> 
- 원자성
    - 하나의 트랜잭션 결과가 모두 성공하거나 모두 실패하는 성질
    - 트랜잭션이 하나의 단위로 처리되는 것
- 일관성
    - 트랜잭션 전후로 데이터베이스가 일관된 상태를 유지하는 성질
    - 일관된 상태 == 데이터베이스가 지켜야 하는 일련의 규칙들을 지키는 상태
    - 데이터베이스가 트랜잭션 이후, 새로운 일관된 상태로 전이될 수 있음.
        - 단, 저장된 데이터들은 모두 새로운 일관된 상태를 유지해야 함.
    
    ![image.png](attachment:cf06799b-666d-4d84-b37a-df53d135e499:image.png)
    
- 격리성
    - 동시에 수행되는 여러 트랜잭션이 서로 간섭하지 않도록 보장하는 성질
    - 한 트랜잭션이 어떤 데이터에 접근하여 조작 중일 땐, 다른 트랜잭션이 접근 불가
- 지속성
    - 트랜잭션이 성공적으로 완료된 후에 그 결과가 영구적으로 반영되는 성질
    - 시스템 장애가 발생하더라도 완료된 트랜잭션의 결과는 손실되지 않아야 함.
