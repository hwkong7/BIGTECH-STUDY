# 파일과 디렉터리

## 파일

### 구성 요소

- 파일의 이름
- 파일을 실행하기 위한 정보
- 파일과 관련한 부가 정보
  - 속성(attribute)또는 메타데이터(metadata)라고 부른다.
- 파일 속성
  - 파일의 형식, 위치, 크기 등 파일과 관련한 다양한 정보가 포함되어 있다.

### 메타데이터 예시

- 파일 크기
- 생성 시간 / 수정 시간
- 위치
- 접근 권한

# 파일 디스크립터 (File Descriptor)

- OS가 파일을 관리하기 위해 사용하는 **정수 값**

## 특징

- 파일 열면 OS가 번호를 줌
- read/write/close 시 사용

## 기본 값

- 0 → stdin (입력)
- 1 → stdout (출력)
- 2 → stderr (에러)

---

# 디렉터리 (Directory)📁

- 파일을 묶어서 관리하는 구조 (폴더)

## 특징

- 트리 구조 (계층 구조)
- 최상위 디렉터리 = 루트(`/`)
- 경로(path)로 위치 표현

## 경로 종류

- 절대 경로: `/home/user/a.txt`
- 상대 경로: `./a.txt`

---

# 디렉터리 엔트리

- 디렉터리는 내부적으로 **테이블 형태**
- 각 파일에 대한 정보 저장

## 포함 정보

- 파일 이름
- 파일 위치 (아이노드 or 블록 주소)

## 파일 할당 방식 (File Allocation)

운영체제가 보조기억장치의 **블록(Block)** 단위로 파일을 저장하는 방법입니다.

| **방식**                | **설명**                                                              | **특징**                                                     |
| ----------------------- | --------------------------------------------------------------------- | ------------------------------------------------------------ |
| **연결 할당 (Linked)**  | 각 블록의 일부에 다음 블록의 주소를 저장하여 연결 리스트 형태로 관리. | 외부 단편화 해결, 하지만 데이터 손실 시 이후 블록 접근 불가. |
| **색인 할당 (Indexed)** | 파일의 모든 블록 주소를 **색인 블록(Index Block)** 하나에 모아 관리.  | 파일 내 임의 접근(Direct Access)이 용이함.                   |

## 아이노드(inode) 기반 파일 시스템

- 리눅스/유닉스 계열(EXT 등)에서 사용하는 대표적인 파일 시스템 구조
- **아이노드(inode):** 파일의 이름만 제외한 모든 메타데이터와 블록 주소를 담고 있는 데이터 구조.
- **EXT4 구조:**
  - **슈퍼 블록:** 파일 시스템의 전체적인 정보(블록 개수 등) 저장.
  - **아이노드 비트맵 / 블록 비트맵:** 어떤 아이노드와 블록이 사용 중인지 표시.
  - **아이노드 테이블:** 실제 아이노드들이 모여 있는 공간.
  - **데이터 블록:** 실제 파일 내용이 저장되는 공간.

## 하드 링크 vs 심볼릭 링크

| **구분**      | **하드 링크 (Hard Link)**                 | **심볼릭 링크 (Symbolic Link)**                     |
| ------------- | ----------------------------------------- | --------------------------------------------------- |
| **원리**      | 원본 파일과 **동일한 아이노드**를 가리킴. | 원본 파일의 **경로(이름)**를 가리키는 별도의 파일.  |
| **삭제 영향** | 원본을 지워도 링크 데이터는 유지됨.       | 원본을 지우면 링크를 사용할 수 없음(Dangling Link). |
| **범위**      | 동일한 파일 시스템 내에서만 가능.         | 다른 파일 시스템(파티션) 간에도 가능.               |

## 마운트 (Mount)

- 어떤 저장장치의 파일 시스템에서 다른 저장장치의 파일 시스템으로 접근할 수 있도록 파일 시스템을 편입시키는 작업
- **예시:** 리눅스에서 `/dev/sda` 장치를 `/mnt/test` 디렉터리에 연결하여 해당 장치의 파일을 사용함.
