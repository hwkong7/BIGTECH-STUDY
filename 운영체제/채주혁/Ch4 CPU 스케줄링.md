# CPU 스케줄링이란?

> 다양한 프로세스와 스레드에 cpu의 사용을 배분하면서 cpu 자원을 관리
> 

프로그램 실행 → 메모리에 프로세스 로드 → 프로세스에 있는 명령어, 데이터 수행하기 위해 자원 배분

**✅ CPU 스케줄러**

이 스케줄링 알고리즘을 결정하고 수행하는 운영체제의 일부분

![IMG_1989.PNG](attachment:7bbaa15f-ed65-4a64-a612-00b1daae06f8:IMG_1989.png)

## 우선순위

> 모든 프로세스는 CPU의 자원을 필요

운영체제는 공정하고 합리적인 방법으로 CPU의 자원을 프로세스에 할당해야함
> 
- 프로세스별 우선순위를 판단하여 PCB에 명시하기도 함
- 대표적인 요소로 CPU 활용률을 고려한다
    - 전체 CPU의 가동 시간 중 작업을 처리하는 시간의 비율
    - 높은 CPU 활용률을 유지하기 위해 입출력이 많은 프로세스의 우선순위를 높게 유지
- 입출력 집중 프로세스를 빨리 실행 → 이후 CPU 집중 프로세스에 할당

## 스케줄링 큐

> 자원을 이용하고 싶다면 줄을 서라!
> 
- 스케줄링 큐를 통해 구현
- 준비 큐
    - CPU를 이용하고 싶은 프로세스의 PCB
- 대기 큐
    - 대기 상태에 접어든 프로세스의 PCB
- 실행되는 프로세스가 할당받은 시간을 소모할 경우 준비 큐로 이동
- 실행 도중 입출력 작업을 수행하는 등 → 대기 큐로 이동

## 선점형 스케줄링과 비선점형 스케줄링

✅ 선점형 스케줄링

운영체제가 프로세스로부터 CPU 자원을 강제로 빼앗아 다른 프로세스에 할당할 수 있는 스케줄링

- CPU 독점을 막고 여러 프로세스에 골고루 자원을 분배
- 문맥 교환 과정에서 오버헤드

✅ 비선점형 스케줄링

프로세스가 종료되거나 스스로 대기 상태에 접어들기 전까지 다른 프로세스가 끼어들 수 없는 스케줄링

- 문맥 교환 횟수가 적어 상대적으로 오버헤드가 적음
- 우선 순위가 높아도 기다려야 함

# 스케줄링 알고리즘

## 선입 선처리 스케줄링

![IMG_1990.PNG](attachment:06dd5dd0-4eac-4e40-8e8c-05bc8ef05e48:IMG_1990.png)

- 비선점 스케줄링
- 선입 선출
- 스레드 우선순위 없음
- 기아 발생하지 않음
- 긴 스레드가 cpu를 오래 사용하면, 늦게 도착한 짧은 스레드 오래 대기

## 최단 작업 우선 스케줄링

![IMG_1991.PNG](attachment:061d974f-b1a1-4063-80b2-065de4aecb5e:IMG_1991.png)

- 비선점 스케줄링
- 실행 시간이 가장 짧은 스레드 선택
- 기아 발생 가능 → 짧은 스레드가 오면 긴 스레드는 실행 기회를 언제 얻을지 예측할 수 없음

## 라운드 로빈 스케줄링

![IMG_1994.PNG](attachment:234e25f4-1cc2-498c-aeb7-013b194cb722:IMG_1994.png)

- 타임 슬라이스 주기로 돌아가면서 선택
- 선점 스케줄링
- 잦은 스케줄링으로 오버헤드가 큼, 특히 타임 슬라이스가 작을 때

## 최소 잔여시간 우선 스케줄링

![IMG_1993.PNG](attachment:069e8f0c-686c-4ae3-9a41-1294b7ec97a5:IMG_1993.png)

## 우선순위 스케줄링

- 우선 순위에 따라 가장 높은 순위의 스레드를 선택
- 선점 스케줄링
    - 더 높은 우선순위가 오면 현재 스레드 강제 중단
- 비선점 스케줄링
    - 현재 실행 중인 스레드가 종료될 때 스케줄링

## 다단계 큐 스케줄링

![IMG_1995.PNG](attachment:28d5d338-fc30-4b48-8708-fb91cfe38cc0:IMG_1995.png)

- 스레드들을 n개의 우선순위 레벨로 구분하여 레벨이 높은 스레드를 우선 처리
- 스레드는 도착 시 우선순위에 따라 해당 레벨 큐에 삽입, 다른 큐로 이동할 수 없음
- 가장 높은 순위의 큐가 빌 때, 다음 순위의 큐에서 스케줄링
- 선점, 비선점 모두 가능

## 다단계 피드백 큐 스케줄링

![IMG_1996.PNG](attachment:2ea24577-87ca-4e9d-b497-488983479dbe:IMG_1996.png)

- 여러 레벨의 큐 사이에 스레드 이동 가능하도록 설계
- 큐마다 스레드가 머무를 수 있는 큐 타임슬라이스가 있음 → 낮은 레벨의 큐일수록 더 긴 타임 슬라이스
- 큐 타임슬라이스 초과 → 아래 큐로 이동, 자발적 중단일 경우 현재 큐 끝에 삽입
- I/O로  실행이 중단될 경우 I/O가 끝나면 동일 레벨 큐 끝에 삽입
- 큐에 있는 시간이 오래되면 기아를 막기 위해 하나 위 레벨 큐로 이동
