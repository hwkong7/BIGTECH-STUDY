# 파일과 디렉터리

## 파일

- 파일의 이름, 실행 정보, 부가 정보로 구성
- 응용 프로그램은 임의로 파일을 할당받아 조작하고 저장할 수 없고, 시스템 콜을 이용해야함

✅ 파일 디스크럽터

프로세스가 할당을 받아 사용 중인 파일을 구분하기 위해 사용

- 저수준에서 파일을 식별하는 정보로, 0 이상의 정수 형태
- 프로세스가 새로 파일을 열거나 생성할 때 프로세스에 할당
- open()
    - 파일을 여는데 사용하고, 연 파일의 파일 디스크립터 반환
- write()
    - 파일 입출력
- close()
    - 파일을 닫는 시스템

## 디렉터리

> 여러 파일들을 관리하기 위해 사용

트리 구조
> 
- 루트 디렉터리
    - 슬래시로 표현
- 경로
    - /this/is/cs

✅ 디렉터리 엔트리

디렉터리에 속한 요소의 관련 정보는 테이블의 형태로 표현되며, 테이블 형태로 표현된 정보의 행 

![image.png](attachment:b4aa1333-f9ea-41d9-bc86-8a4d18b59cd7:image.png)

![image.png](attachment:eb2cc9f0-d6a9-4c97-8df4-d25e5cc862c8:image.png)

- 파일의 이름과 파일이 저장된 위치를 유추할 수 있는 정보가 있음
- 보조기억장치에 저장되어 있는 위치를 알 수 있어 디렉터리에 속한 파일의 위치를 읽어 실행하거나 디렉터리에 속한 다른 디렉터리의 위치를 찾아 이동
- 생성시간, 수정된 시간, 크기도 명시하기도 함

## 파일 할당

✅ 블록

운영체제가 파일과 디렉터리를 읽고 쓰는 단위

- 4096 바이트
- 하나의 파일이 보조기억장치에 저장될 때 블록을 할당받아 저장

✅ 연결 할당

각 블록의 일부에 다음 블록의 주소를 저장하여 다음 블록을 가리키는 형태

- 파일 이름, 파일을 이루는 첫 번째 블록 주소, 블록 단위의 길이

✅ 색인 할당

파일을 이루는 모든 블록의 주소를 색인 블록이라는 특별한 블록에 모아 관리

- 파일 이름과 함께 색인 블록 주소가 명시
- 색인 블록만 알면 접근하고자 하는 파일 데이터에 접근

# 파일 시스템

파일 시스템의 종류는 다양

운영체제마다 각기 다른 파일 시스템, 같더라도 다른 파일 시스템을 사용하거나,

하나의 컴퓨터에 여러 파일 시스템을 사용

## 아이노드 기반 파일 시스템

> 파일이 저장된 위치와 속성이 담긴 색인 블록

파일마다 각각의 아이노드를 가지고, 아이노드에 각각의 번호가 부여
> 

![image.png](attachment:f862bde8-30f2-4e78-8f1c-b1460df63a1d:image.png)

- 부트 블록
    - 실질적인 파일 데이터 저장 x
    - 부팅과 파티션 관리를 위한 특별한 정보가 모여 있는 영역
- 슈퍼 블록
    - 아이노드의 총 개수, 총 블록 개수, 블록 크기 등
- 그룹 식별자
    - 블록 그룹에 대한 메타데이터
- 블록 비트맵
    - 현재 블록 그룹 내에서 데이터가 어떻게 할당되었는지 저장
- 아이노드 비트맵
    - 현재 블록 그룹 내에서 아이노드가 어떻게 할당되었는지
- 아이노드 테이블
    - 각 파일의 아이노드 정보를 저장
- 데이터 블록
    - 각 파일의 데이터 저장

## 마운트

> 저장 장치의 파일 시스템에서 다른 저장장치의 파일 시스템으로 접근할 수 있도록 편입
> 

![image.png](attachment:05842b0e-633d-4173-ad87-2954b0be3bef:image.png)

- USB를 통해 접근할 수 있는 이유 → USB 메모리의 파일 시스템이 데스크톱 컴퓨터의 파일 시스템에 마운트 되었기 때문
- /mnt 경로로 마운트하면 USB 메모리의 파일 시스템은 /mnt 경로로 마운트
