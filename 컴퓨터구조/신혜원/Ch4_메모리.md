# 메모리

- 실행 중인 프로그램은 메모리에 저장되며, CPU는 메모리에 저장된 정보를 읽고 쓰며 실행한다.

## RAM(Random Access Memory)

- 메인 메모리로 주로 사용되는 하드웨어
- **휘발성 저장장치(volatile memory)**
  - 전원이 꺼지면 저장된 데이터가 **사라진다.**
- **↔ 비휘발성 저장장치(non-volatile memory)**
  - 전원이 꺼져도 저장된 내용이 **유지된다.**
  - 보조기억장치
- CPU가 실행할 프로그램과 데이터를 저장하는 공간

### RAM의 역할

- CPU는 보조기억장치에 저장된 프로그램을 곧바로 실행할 수 없다.
- 따라서 실행할 프로그램은 보조기억장치에서 RAM으로 복사되어야 한다.
- RAM 용량이 작으면 보조기억장치에서 데이터를 자주 가져와야 하므로 실행 시간이 길어질 수 있다.
- RAM 용량이 충분하면 여러 프로그램을 동시에 실행하는 데 유리하다.

### 임의 접근(random access)

- 저장된 요소에 순차적으로 접근하지 않고 **원하는 위치에 바로 접근할 수 있는 방식**
- 1번지에 접근하는 시간과 1000번지에 접근하는 시간이 거의 동일하다.

![image.png](attachment:b00958fc-ec4d-40a0-8ff9-c54537e3a9cc:e83da97a-61d2-4cc0-909a-8c1ee9ddacf1.png)

### 순차 접근(sequential access)

- 특정 위치에 저장된 요소에 접근하기 위해 **처음부터 순서대로 접근하는 방식**
- 접근 위치에 따라 시간이 달라질 수 있다.

![image.png](attachment:38a1857a-4571-48cc-add7-3f1fec92ba57:35036652-447f-454d-a58d-d195ea61b7d5.png)

## RAM의 종류

### DRAM(Dynamic RAM)

- 시간이 지나면 저장된 데이터가 **점차 사라지는 RAM**
- 데이터 소멸을 막기 위해 일정 주기로 재활성화(재저장, refresh)해야 한다.
- 장점
  - 소비 전력이 낮음
  - 가격이 저렴함
  - 집적도가 높아 대용량 설계에 유리함
- 일반적으로 주기억장치로 사용된다.

### SRAM(Static RAM)

- 시간이 지나도 저장된 데이터가 **사라지지 않는 RAM**
- 단, 전원이 꺼지면 데이터는 사라지므로 비휘발성 저장장치는 아니다.
- 장점
  - DRAM보다 빠름
- 단점
  - 소비 전력이 큼
  - 가격이 비쌈
  - 집적도가 낮음
- 대용량 메모리보다는 속도가 중요한 캐시 메모리 등에 사용된다.

### SDRAM(Synchronous Dynamic RAM)

- 클럭 신호와 동기화된 DRAM
- 클럭에 맞춰 CPU와 정보를 주고받을 수 있다.

### DDR SDRAM(Double Data Rate SDRAM)

- SDRAM보다 대역폭을 넓혀 속도를 빠르게 만든 메모리
- 한 클럭당 한 번 전달하는 SDRAM과 달리, 한 클럭당 두 번 데이터를 주고받을 수 있다.
- DDR2, DDR3, DDR4 등으로 갈수록 대역폭이 더 넓어진다.

# 캐시 메모리

- CPU는 메모리에 자주 접근해야 한다.
- 하지만 메모리 접근 속도는 CPU가 레지스터에 접근하는 속도보다 느리다.
- 이 속도 차이를 줄이기 위해 사용하는 저장장치가 캐시 메모리이다.

## 캐시 메모리란

- CPU의 연산 속도와 메모리 접근 속도의 차이를 줄이기 위한 저장장치
- CPU와 메모리 사이에 위치한 **SRAM 기반 저장장치**
- CPU가 자주 사용할 일부 데이터를 미리 가져와 저장한다.

![image.png](attachment:f8e074df-a7f8-4b8c-a795-b95a5e8fc1dd:image.png)

## 캐시 레벨

### L1 캐시 (Level 1 cache)

- 코어와 가장 가까움
- 가장 빠르지만 크기가 가장 작음

### L2 캐시 (Level 2 cache)

- L1보다 멀고, 속도는 조금 느리지만 크기는 더 큼

### L3 캐시 (Level 3 cache)

- 여러 코어가 공유하는 경우가 많음
- 가장 크지만 가장 느림

![image.png](attachment:aa7c8dc3-c731-42af-8373-6426ede90560:image.png)

#### 크기와 속도 관계

- 크기: L1 < L2 < L3
- 속도: L3 < L2 < L1

## 캐시 히트와 캐시 미스

### 캐시 히트(cache hit)

- CPU가 필요한 데이터를 캐시 메모리에서 찾은 경우
  ![image.png](attachment:d84504ec-5f4f-49aa-ab41-cfcadb899d97:ec11a8d9-943f-42a0-b184-8cb120697b02.png)

### 캐시 미스(cache miss)

- CPU가 필요한 데이터를 캐시에서 찾지 못해 메모리까지 접근해야 하는 경우
  ![image.png](attachment:84b114e8-2294-4cf4-9656-135d7c5ccd15:89bc5bad-b503-4165-8d6e-56aff8b40545.png)

## 캐시 적중률

- 캐시가 히트되는 비율
- 공식

$$
캐시\ 히트\ 횟수 \div (캐시\ 히트\ 횟수 + 캐시\ 미스\ 횟수)
$$

## 참조 지역성의 원리

- 캐시 메모리는 CPU가 자주 사용할 법한 데이터를 예측해 저장한다.
- 이때 사용하는 핵심 개념이 참조 지역성(locality of reference)이다.

### 시간 지역성(temporal locality)

- CPU는 최근에 접근했던 메모리 공간에 다시 접근하려는 경향이 있다.
- ex) 변수 값 재사용

### 공간 지역성(spatial locality)

- CPU는 접근한 메모리 공간의 근처에 접근하려는 경향이 있다.
- ex) 배열을 순차적으로 탐색

### 공간 지역성 예시

- 2차원 배열을 순차적으로 접근하는 코드가
- 비순차적으로 접근하는 코드보다 실제 실행 속도가 더 빠를 수 있다.
- 이유는 메모리에 연속 저장된 데이터를 순서대로 접근하기 때문이다.

# 캐시 메모리의 쓰기 정책과 일관성

- CPU가 캐시 메모리에 데이터를 쓸 때는
- 캐시 메모리와 메모리 사이 데이터 일관성을 유지해야 한다.

![image.png](attachment:983da12c-6886-4bd4-a05d-458db6086fbd:image.png)

![image.png](attachment:53ee9498-7789-42bd-8a80-bc86088963fa:image.png)

## 즉시 쓰기(Write-through)

- 데이터를 쓸 때 캐시 메모리와 메모리에 동시에 반영하는 방식
- 장점
  - 일관성 유지가 쉬움
- 단점
  - 데이터를 쓸 때마다 메모리에도 접근해야 하므로 성능상 불리할 수 있음

![image.png](attachment:4e5fb7d6-7ccb-4699-90cc-389d9d4b5419:4ade322d-c2a1-4e3c-ad7b-3601a5a8101d.png)

## 지연 쓰기(Write-back)

- 우선 캐시 메모리에만 값을 쓰고
- 나중에 수정된 데이터를 한 번에 메모리에 반영하는 방식
- 장점
  - 메모리 접근 횟수를 줄여 속도가 빠를 수 있음
- 단점
  - 캐시와 메모리 간 불일치 가능성이 있음

![image.png](attachment:883af272-0768-4ccd-bb1a-f1830d3ab6e5:49297382-752d-4eda-a4c1-7a5ee3f306fe.png)

## 멀티코어 환경의 일관성 문제

- 여러 코어가 각자 캐시 메모리를 가지면
- 코어 간 캐시 내용이 서로 달라질 수 있다.
- 이를 해결하기 위해 캐시 일관성 프로토콜이 사용된다.
