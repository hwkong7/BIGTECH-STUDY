# 5. 보조기억장치와 입출력장치

# 보조기억장치

- 메모리의 휘발성을 보완하고 더 큰 저장 공간을 제공하는 장치

## 대표적인 보조기억장치

### 하드 디스크(HDD)

- 자기적인 방식으로 데이터를 읽고 쓰는 보조기억장치
- 플래터와 헤드를 이용해 데이터를 저장하고 접근한다.

### 플래시 메모리 기반 저장장치

- 전기적인 방식으로 데이터를 읽고 쓰는 반도체 기반 저장장치
- USB 메모리, SD 카드, SSD 등이 여기에 해당한다.
- 오늘날 보조기억장치로는 SSD가 많이 사용된다.

## 보조기억장치의 본질

- 전원이 꺼져도 데이터를 안전하게 보관한다.
- CPU가 필요로 하는 정보를 빠르게 메모리에 전달해야 한다.

# RAID(Redundant Array of Independent Disks)

- 여러 개의 독립적인 보조기억장치를 마치 하나의 보조기억장치처럼 사용하는 기술

![image.png](attachment:7a1c7141-91b3-4ae0-adfd-9f3bf1093579:image.png)

- 목적
  - 데이터 안정성 확보
  - 성능 향상

## RAID 0

- 데이터를 여러 저장장치에 **나누어 저장**하는 방식
- 스트라이핑(striping)을 사용한다.

### 장점

- 입출력 속도가 빠름

### 단점

- 저장된 정보가 안전하지 않음
- 디스크 하나라도 고장 나면 전체 데이터가 손상될 수 있음

![image.png](attachment:1e792b11-24e2-42b3-86fc-7c4dbbe84359:image.png)

## RAID 1

- 완전한 복사본을 만들어 저장하는 방식
- 미러링(mirroring)이라고도 한다.

### 장점

- 복구가 간단함
- 안정성이 높음

### 단점

- 원본과 복사본 두 곳에 모두 써야 하므로 쓰기 속도가 느릴 수 있음
- 사용 가능한 용량이 줄어든다.

![image.png](attachment:841f8d97-7b7a-498f-ad23-359d018d91bf:image.png)

## RAID 4

- 패리티 정보를 저장하는 디스크를 따로 두는 방식

### 장점

- RAID1보다 적은 수의 디스크로도 데이터 보호 가능

### 단점

- 패리티 디스크에 병목 현상이 발생할 수 있음

![image.png](attachment:76049e51-0a60-443f-bba8-dd077f3c12f9:image.png)

## RAID 5

- 패리티를 분산 저장하는 방식
- RAID4의 병목 현상을 완화한 방식

![image.png](attachment:3a8183c0-30ef-4c2b-9089-ac66d5c027e6:image.png)

## RAID 6

- RAID5와 비슷하지만 서로 다른 2개의 패리티를 두는 방식
- RAID4, RAID5보다 안정성이 높다.
- 대신 패리티를 더 많이 저장하므로 쓰기 속도는 일반적으로 더 느리다.

![image.png](attachment:ed54f5e6-eb21-43b4-a189-3b358b1f6981:image.png)

## Nested RAID

- 여러 RAID 레벨을 혼합한 방식
- ex)
  - RAID10 = RAID0 + RAID1
  - RAID50 = RAID0 + RAID5
  ![image.png](attachment:9dd1bf1c-54de-432a-aa11-86f45e71dc45:image.png)

# 입출력 기법

- 보조기억장치를 포함한 다양한 입출력장치들이 컴퓨터 내부와 정보를 주고받는 방식

## 장치 컨트롤러와 장치 드라이버

### 장치 컨트롤러(device controller)

- CPU와 입출력장치 사이의 통신을 중개하는 하드웨어
- CPU는 입출력장치와 직접 연결되어 정보를 주고받지 않고
- 장치 컨트롤러를 통해 소통한다.

### 장치 드라이버(device driver)

- 장치 컨트롤러의 동작을 알고
- 장치 컨트롤러가 컴퓨터 내부와 정보를 주고받을 수 있도록 하는 프로그램
- 운영체제가 기본 드라이버를 제공하는 경우도 많지만
- 일부 장치는 별도 드라이버 설치가 필요하다.

# 프로그램 입출력

- 프로그램 속 명령어로 입출력 작업을 수행하는 방식
- CPU가 입출력 명령어를 실행하여 장치 컨트롤러와 상호작용한다.

## 프로그램 입출력의 두 종류

### 고립형 입출력(Isolated I/O)

- 입출력장치를 위한 주소 공간과 메모리를 위한 주소 공간을 별도로 구분하는 방식
- 입출력 전용 명령어가 필요하다.

### 메모리 맵 입출력(Memory Mapped I/O)

- 입출력장치 주소 공간과 메모리 주소 공간을 구분하지 않는 방식
- 메모리 접근 명령어로 입출력도 가능하다.

# 인터럽트 기반 입출력: 다중 인터럽트

- 여러 입출력장치에서 인터럽트가 동시에 발생할 수 있다.
- 이때 CPU는 여러 인터럽트를 처리해야 한다.

![image.png](attachment:01827b58-814d-4827-94a7-2bd210e32f9f:image.png)

## 다중 인터럽트 처리

- 인터럽트 **우선순위가 더 높은 것**이 **먼저** 처리되는 경우가 일반적이다.
- 현재 처리 중인 인터럽트보다 **더 높은 우선순위**의 인터럽트가 들어오면
  - 현재 작업을 잠시 멈추고 더 높은 우선순위 인터럽트를 먼저 처리한다.
- **우선순위가 낮다면**
  - 현재 인터럽트 처리가 끝난 뒤 나중에 처리한다.
  ![image.png](attachment:2a4c257a-5327-4072-aaf0-8d6d0d503438:image.png)

## NMI(Non-Maskable Interrupt)

- 인터럽트 비트를 비활성화해도 무시할 수 없는 인터럽트
- 매우 높은 우선순위를 가진다.

## PIC(Programmable Interrupt Controller)

- 여러 장치 컨트롤러로부터 들어오는 하드웨어 인터럽트 요청의 우선순위를 판단해
- CPU에게 지금 처리해야 할 인터럽트를 알려주는 하드웨어
- 일반적으로 많은 인터럽트를 관리하기 위해 2개 이상의 계층으로 구성된다.
- 단, NMI까지 판단하지는 않는다.

![image.png](attachment:4305bdd1-8dc1-4919-b679-30a2b32b9c3c:image.png)

# DMA(Direct Memory Access) 입출력

- CPU를 거치지 않고 입출력장치와 메모리 사이에서 직접 데이터를 주고받게 하는 방식
- 공통점
  - CPU가 입출력장치와 메모리간의 데이터 이동을 주도해야 하며, 이동하는 데이터들도 반드시 CPU를 거친다는 점
- 입출력장치의 데이터를 메모리에 저장하는 경우
  1. 장치 컨트롤러로부터 데이터를 하나씩 읽어 레지스터에 적재하고,
  2. 적재한 데이터를 하나씩 메모리에 저장해야 한다.

![image.png](attachment:a66a0963-ec2d-4df9-9c83-7cdffc64b3c2:image.png)

## DMA 동작 과정

1. CPU가 DMA 컨트롤러에게 입출력장치 주소, 수행할 연산, 메모리 주소 등의 정보를 주며 입출력 작업을 명령한다.
2. DMA 컨트롤러가 CPU 대신 장치 컨트롤러와 상호작용하며 입출력 작업을 수행한다.
3. 작업이 끝나면 DMA 컨트롤러가 CPU에게 인터럽트를 걸어 완료를 알린다.

![image.png](attachment:c3d652a9-aa91-4ce8-997f-057c3212a64e:image.png)

## DMA의 장점

- 입출력 과정에서 CPU 부담을 크게 줄일 수 있다.
- 입출력장치와 메모리 사이 데이터가 CPU를 거치지 않으므로 효율적이다.

## 사이클 스틸링(Cycle Stealing)

- DMA 컨트롤러도 시스템 버스를 사용해 메모리에 직접 접근한다.
- 시스템 버스는 동시에 사용할 수 없기 때문에
- DMA가 버스를 사용할 때 CPU는 잠시 시스템 버스를 사용하지 못한다.
- 이렇게 DMA가 CPU의 버스 사용 기회를 조금씩 가져가는 현상을 사이클 스틸링이라고 한다.

# PCIe 버스

- 대표적인 입출력 버스 중 하나
- SSD, GPU, 네트워크 인터페이스 카드 등 다양한 장치를 연결할 수 있다.
- 오늘날 메인보드에서 매우 널리 사용된다.

![image.png](attachment:13a26ee6-21f6-47c2-97f2-6b80f1dac77f:image.png)

## PCIe 성능과 관련해 알아둘 점

### 1. 버전

- PCIe는 계속 발전해 왔으며 버전에 따라 최대 속도가 달라진다.
- ex)
  - PCIe 1.0
  - PCIe 2.0
  - PCIe 3.0
  - PCIe 4.0
  - PCIe 5.0
  - PCIe 6.0
  - PCIe 7.0

### 2. 레인(lane)

- PCIe 버스를 통해 정보를 송수신하는 단위
- 레인이 2개, 4개, 8개가 되면 동시에 더 많은 양을 주고받을 수 있다.
- 보통 x1, x4, x8, x16처럼 표기한다.
- ex) PCIe 4.0 x4
  - PCIe 4.0 버전
  - 4개의 레인 사용

# GPU의 용도와 처리 방식

## GPU란

- Graphics Processing Unit
- 원래는 그래픽 연산을 위해 탄생한 장치
- 현재는 딥러닝, 가상화폐 채굴 등 다양한 분야의 연산까지 수행한다.

## GPGPU

- General-Purpose computing on Graphics Processing Units
- 그래픽 처리 외 범용 연산 목적으로 GPU를 사용하는 기술

## GPU의 특징

- 개별 코어 성능은 CPU 코어보다 떨어지는 편이지만
- 수백 개에서 수천 개의 코어를 포함할 수 있다.
- 따라서 단순하고 반복적인 작업을 병렬로 빠르게 처리하는 데 유리하다.
- 자체 캐시 메모리와 큰 메모리 대역폭을 가진다.

## 병렬 처리(parallel processing)

- 하나의 크고 복잡한 문제를
- 여러 개의 작고 단순한 문제로 나눈 뒤
- 여러 코어가 동시에 처리하는 방식

## GPU와 CPU의 차이

### GPU

- 단순한 연산을 빠르게, 병렬적으로 수행하는 데 특화
- 많은 코어를 동원해 대량 작업 처리에 유리
- CPU의 산술 연산을 보조하는 보조프로세서(coprocessor)처럼 볼 수 있다.

### CPU

- 범용적인 연산 수행에 적합
- 복잡한 제어와 다양한 작업 처리에 강함

## GPU가 CPU를 완전히 대체하지 못하는 이유

- GPU 개별 코어는 CPU보다 느리고 기능도 단순하다.
- 운영체제 실행이나 복잡한 명령어 처리 등은 CPU가 더 적합하다.
- 즉, GPU와 CPU는 경쟁 관계라기보다 역할이 다른 관계이다.

# CUDA

- 엔비디아(NVIDIA)가 개발한 프로그래밍 모델
- 일반적인 프로그래밍 언어를 통해 GPU가 수행할 작업을 작성할 수 있게 해 준다.

## CUDA 코드 구성

### 호스트 코드(host code)

- CPU가 실행할 코드

### 디바이스 코드(device code)

- GPU가 실행할 코드

## 실행 흐름

- CPU가 호스트 코드를 실행하다가
- GPU가 수행할 디바이스 코드를 만나면
- GPU 메모리로 관련 코드와 작업이 전달되고
- GPU가 이를 실행한 뒤 결과를 CPU에 알려준다.
