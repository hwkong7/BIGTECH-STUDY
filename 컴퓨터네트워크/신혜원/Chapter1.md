# 1. 네트워크의 큰 그림
## - 네트워크의 기본 구조
- 네트워크는 그래프 형태로 표현할 수 있다.
  - 노드(Node): 호스트, 서버, 라우터, 스위치와 같은 네트워크 장비
  - 간선(Edge): 유선 또는 무선 통신 매체
  - 노드랑 노드 사이 연결구조 = 네트워크 토폴로지
   - 망형, 트리형, 링형, 성형, 버스형(선형)
  - 호스트(Host)는 네트워크의 가장자리에 위치하며,
  데이터를 송신하거나 수신하는 주체가 되는 장치이다.
  - 1) 요청 - 클라이언트
  - 2) 응답 - 서버
## - LAN과 WAN
### LAN (Local Area Network)
- 비교적 좁은 범위의 네트워크
> ex) 가정, 학교, 사무실 내부 네트워크

### WAN (Wide Area Network)
- 넓은 지역을 연결하는 네트워크
- 여러 LAN을 연결하는 형태
- 일반적으로 ISP가 구축 및 운영
> ex) 인터넷

 Q) 100GB 파일 한번에 전송가능? 
 > A) X => 여러개로 쪼개짐
 
 => 패킷 단위로 쪼개서 송수신 -> 재조립
### 패킷
- 네트워크에서 데이터를 주고받을 때 사용하는 전송 단위이다.
- 일반적으로 다음과 같이 구성된다.
  - 헤더(Header): 주소, 제어 정보 등 부가 정보
  - 페이로드(Payload): 실제로 전달할 데이터
  - 트레일러(Trailer): 일부 프로토콜에서 오류 검출 등에 사용
## 주소의 개념과 전송 방식

### 주소
- 패킷의 헤더에 명시되는 정보
- 예: IP 주소, MAC 주소

### 전송 방식

| 방식 | 설명 |
|---|---|
| 유니캐스트 | 송신지와 수신지가 일대일로 메시지를 주고받는 전송 방식 |
| 브로드캐스트 | 네트워크상의 모든 호스트에게 메시지를 전송하는 방식 (전송 범위: 브로드캐스트 도메인) |
| 멀티캐스트 | 네트워크 내 동일 그룹에 속한 호스트에게만 전송하는 방식 |
| 애니캐스트 | 동일 그룹에 속한 호스트 중 가장 가까운 호스트에게 전송하는 방식 |
## - 두 호스트가 패킷을 주고받는 과정
### - 프로토콜(Protocol)
- : 네트워크에서 통신을 수행하기 위한 노드 간의 약속된 규칙과 절차이다.
### - 호스트와 네트워크 장비 -> 같은 프로토콜 이해&통신
  => 프로토콜은 흔히 네트워크의 언어와 같다
  - 프로토콜마다 목적과 특징이 다르다
  > ex) - IP: 네트워크 간 주소 지정과 패킷 전달을 담당
  >     - ARP: IP 주소를 MAC 주소에 대응
  >     - TCP: 신뢰성 있는 전송 제공
  >     - UDP: 빠르고 단순한 전송 제공
  >     - HTTP/HTTPS: 웹 통신에 사용되는 프로토콜
### - 네트워크 참조 모델
  - 호스트 간에 패킷을 주고받는 과정 ≒ 택배를 주고받는 과정
  : 통신이 이루어지는 단계를 계층적으로 표현한 것
  송신 ) 상위 -> 하위 | 수신) 하위 -> 상위
### - OSI모델 : 국제 표준화 기구에서 만든 네트워크 참조 모델

| 계층 | 설명 |
|---|---|
| 응용 계층 (Application Layer) | 사용자와 가장 밀접하게 맞닿아 있으며 여러 네트워크 서비스를 제공하는 계층 |
| 표현 계층 (Presentation Layer) | 번역기와 같은 역할을 하는 계층으로 데이터 인코딩, 압축, 암호화를 담당 |
| 세션 계층 (Session Layer) | 응용 프로그램 간 연결 상태를 의미하는 세션을 관리하는 계층 |
| 전송 계층 (Transport Layer) | 데이터 전송 과정에서 발생할 수 있는 유실이나 순서 문제에 대비해 신뢰성 있는 전송을 제공하는 계층 |
| 네트워크 계층 (Network Layer) | 네트워크 간 통신을 가능하게 하며 서로 다른 네트워크 사이에서 패킷을 전달하는 계층 |
| 데이터 링크 계층 (Data Link Layer) | 같은 LAN 내 호스트 간 올바른 데이터 전달을 담당하며 MAC 주소를 사용 |
| 물리 계층 (Physical Layer) | OSI 모델의 최하위 계층으로 전기적·물리적 신호를 유무선 통신 매체를 통해 전달하는 계층 |

### - TCP/IP 모델

| TCP/IP 계층 | OSI 모델 대응 계층 |
|---|---|
| 응용 계층 (Application Layer) | 세션 계층 + 표현 계층 + 응용 계층 |
| 전송 계층 (Transport Layer) | 전송 계층 |
| 인터넷 계층 (Internet Layer) | 네트워크 계층 |
| 네트워크 액세스 계층 (Network Access Layer) | 데이터 링크 계층 + 물리 계층 |

※ 네트워크 액세스 계층은 링크 계층(Link Layer) 또는 네트워크 인터페이스 계층(Network Interface Layer)이라고도 한다.
## - 캡슐화와 역캡슐화 
### - 캡슐화
: 송신 과정에서 헤더(및 트레일러)를 추가해 나가는 과정
### - 역캡슐화
: 캡슐화 과정에서 붙인 헤더(및 트레일러)를 각 계층에서 확인한 뒤 제거하는 과정
※ 패킷을 송신하는 쪽에서는 상위 계층에서 하위 계층으로 정보를 보내고, 패킷을 수신하는 쪽에서는 하위 계층에서 상위 계층으로 정보를 받아들인다.
※ 네트워크 계층 구조를 이용하면 프로토콜을 계층별로 구성할 수 있다.
※ 하나의 패킷은 헤더와 페이로드(때로는 트레일러까지)를 포함하며, 프로토콜의 목적과 특징에 따라 헤더의 내용은 달라질 수 있다.
## 계층별 패킷 이름

| 계층 | 패킷의 이름 |
|---|---|
| 그 이상의 계층 | 데이터(data) 또는 메시지(message) |
| 전송 계층 | TCP 기반 패킷의 경우) 세그먼트(segment) UDP 기반 패킷의 경우) 데이터그램(datagram) |
| 네트워크 계층 | 패킷(이하 IP 패킷) 또는 데이터그램 |
| 데이터 링크 계층 | 프레임(frame) |
| 물리 계층 | 심볼(symbol) 또는 비트(bit) |

## 계층 구조를 사용하는 이유
- 네트워크 기능을 역할별로 나누어 설계할 수 있다.
- 특정 계층의 변경이 다른 계층에 미치는 영향을 줄일 수 있다.
- 문제 발생 시 어느 계층에서 문제가 생겼는지 파악하기 쉽다.

## 인터넷의 기본 구조
- 인터넷은 여러 네트워크가 서로 연결된 네트워크의 집합이다.
- 각 네트워크는 라우터를 통해 서로 연결된다.
