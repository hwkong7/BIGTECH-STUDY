# 2. 물리 계층과 데이터 링크 계층
## 이더넷 
- : LAN 환경에서 사용되는 대표적인 데이터 링크 계층 기술로,
통신 매체를 통해 신호를 송수신하는 방법과 데이터 프레임 형식을 정의한 표준
> IEEE 802.3

- 1️⃣오늘날의 (우선)LAN 대부분이 이더넷 표준을 따르기 때문에 대다수의 LAN 장비들이 특정 이더넷 표준을 따른다.
- 2️⃣이더넷 표준이 달라지면 통신 매체의 종류를 비롯한 신호 송수신 방법, 나아가 최대 지원 속도도 달라질 수 있다.
### - 이더넷 프레임 : 이더넷 기반의 네트워크에서 주고받는 프레임
- (유선) 프레임 ≒ 이더넷 프레임 => LAN은 대부분 이더넷을 기반으로 구현되고 있기 때문
### 1) 프리앰블 
- : 송수신지 동기화를 위해 사용되는 8바이트 크기의 정보로
- 7바이트 : 10101010
- SFD(Start Frame Delimiter) 1바이트 : 10101011
### 2) 송수신지 MAC 주소
: 프레임에서 가장 중요한 정보 , 프레임에는 송신지와 수신지를 특정할 수 있는 6바이트(48비트) 길이의 MAC 주소가 명시되며, 다음과 같이 콜론(:)으로 구분된 12자리 16진수로 구성되어 있습니다.
### ab:cd:ab:cd:00:01
+ 물리적 주소, 네트워크 인터페이스마다 하나씩 부여되는 주소
  - 여기서 네트워크 인터페이스란?
    -네트워크를 향하는 통로, 연결 매체와의 연결 지점을 의미
> NIC가 여러 개인 상황처럼 네트워크 인터페이스가 여럿이라면 한 호스트가 여러 개의 MAC주소를 가질 수 있다.

### 3) 타입/길이
- 타입 : 캡슐화된 상위 계층의 정보를 의미 => 타입을 통해 어떤 상위 계층 프로토콜이 캡슐화되었는지 알 수 있다.
- ex) IP(IPv4)가 캡슐화된 정보를 운반한다면 타입에는 16진수 0800이 명시되고, ARP 프로토콜이 캡슐화된 정보를 운반한다면 타입에는 16진수 0806이 명시된다.

### 4) 데이터
- 데이터 필드에는 페이로드, 즉 상위 계층으로 전달하거나 전달받을 데이터가 명시됩니다.
- ⭐ 데이터 필드에 포함될 수 있는 데이터의 최대 크기가 정해져 있다
  - 최대 크기는 **1500바이트** 이하로 제한(최소 46byte)
    - 이보다 큰 데이터를 보낼 경우에는 여러 패킷으로 나뉘어 보내진다.
> 데이터 필드에 명시 가능한 최대 페이로드의 크기는 일반적으로 1500바이트지만, 더 큰 데이터를 포함할 수 있는 특별한 프레임도 있다. => 점보 프레임(jumbo frame)

### 5) FCS(Frame Check Sequence)
- : 트레일러
- 프레임의 오류가 있는지의 여부를 확인하기 위한 필드
- CRC(Cyclic Redundancy Check)라는 오류 검출용 값이 명시된다.
- 1) 송신지에서 전송할 데이터와 더불어 전송할 데이터에 대한 CRC값을 계산하여 보내면 
- 2) 수신지에서는 전달받은 데이터에 대한 CRC값을 계산해
- 3) 그 값을 전달받은 CRC값과 대조한다.
- 4) 두 값이 같을 경우 프레임에 오류가 없다고 판단

## 유선 통신 매체

### Twisted Pair 케이블
- LAN에서 가장 많이 사용하는 유선 통신 매체
- 구리선 두 가닥을 꼬아서 만든 케이블
- 선을 꼬는 이유 ) 전자기적 간섭(노이즈)을 줄이기 위해서
- 대표적으로 RJ-45 커넥터를 사용한다.

### 종류
UTP : 차폐(shield)가 없는 케이블
STP : 금속 차폐가 있는 케이블

### 케이블 카테고리
- 카테고리는 케이블의 성능과 최대 속도를 구분하는 기준이다.

| 카테고리 | 속도 |
|---|---|
| Cat5 | 100Mbps |
| Cat5e | 1Gbps |
| Cat6 | 1Gbps |
| Cat6a | 10Gbps |
| Cat7 | 10Gbps |
| Cat8 | 40Gbps |

## 무선 통신
### WiFi
- : IEEE 802.11 표준을 따르는 무선 LAN 기술

### WiFi세대
| WiFi   | 표준       |
| ------ | -------- |
| WiFi 4 | 802.11n  |
| WiFi 5 | 802.11ac |
| WiFi 6 | 802.11ax |
| WiFi 7 | 802.11be |

### WiFi 주파수 대역
- WiFi 주로 2.4GHz와 5GHz 대역을 사용한다.
- 최근에는 6GHz 대역(WiFi 6E)도 사용된다.

| 주파수    | 특징                 |
| ------ | ------------------ |
| 2.4GHz | 속도는 느리지만 도달 거리가 길다 |
| 5GHz   | 속도는 빠르지만 도달 거리가 짧다 |

### WiFi 채널
- 같은 주파수 대역을 사용하는 네트워크가 많으면 간섭이 발생한다.
- 2.4GHz에서는 채널이 서로 겹치기 때문에 일반적으로 1, 6, 11 채널을 사용한다.

## AP와 SSID
### AP(Access Point)
- 무선 네트워크에 기기를 연결해주는 장치
- 일반적으로 무선 공유기가 AP 역할을 한다.
### SSID
-  SSID(Service Set Identifier)는 무선 네트워크 이름이다.
-  WiFi 목록에서 보이는 네트워크 바로 SSID이다.

## 네트워크 인터페이스(NIC)
- 컴퓨터를 네트워크에 연결하는 장치
- **특징**
  - 네트워크 송수신 담당
  - 각 NIC에는 MAC 주소가 존재
  - 네트워크 인터페이스 카드 또는 LAN 카드라고도 불림
- NIC는 네트워크 신호를 프레임 형태로 처리하여 컴퓨터와 네트워크 간 통신을 담당한다.

## 네트워크 장비

### 허브(Hub)
- 물리 계층 장비로 여러 컴퓨터를 연결하는 장치
- 허브는 OSI 1계층 장비
- **특징**
  - 수신한 신호를 모든 포트로 전달
  - 데이터 판단 없이 단순 전달
  - 반이중 통신(Half Duplex) 사용
    - : 송신 또는 수신을 번갈아 가면서 수행해야 하는 통신 

### 충돌 도메인(Collision Domain)
- 허브는 모든 장치가 하나의 네트워크처럼 동작하기 때문에
- 여러 장치가 동시에 데이터를 보내면 충돌(Collision)이 발생할 수 있다.
- 이때 충돌이 발생할 수 있는 범위를 충돌 도메인이라고 한다.
- 허브에서는 모든 장치가 하나의 충돌 도메인에 속한다.

### 스위치(Switch)
- 스위치는 허브의 단점을 개선한 데이터 링크 계층 장비이다.
- 스위치는 각 포트가 별도의 충돌 도메인을 가진다.
- **특징**
  - 목적지 포트로만 데이터 전달
  - MAC 주소 기반 동작
  - 전이중 통신(Full Duplex) 지원
    - : 동시 송수신이 가능한 통신 => 콜리전 도메인이 좁다

## 스위치의 MAC 주소 학습
- 스위치는 MAC 주소 테이블을 이용해 데이터를 전달한다.
- **동작 과정**
  1) 프레임이 스위치에 들어옴
  2) 송신자의 MAC 주소를 학습
  3) MAC 주소와 포트를 테이블에 저장
  4) 목적지 MAC 주소 확인
  5) 해당 포트로만 프레임 전달
- 예시
  - 포트 1 → MAC A
  - 포트 3 → MAC C 
  - A → C로 데이터를 보내면
  - → 스위치는 3번 포트로만 데이터 전달
  - MAC 주소를 모르는 경우
  - 목적지 MAC 주소가 테이블에 없으면
  - → 수신 포트를 제외한 모든 포트로 전송 (Flooding)
  - 이후 응답을 통해 MAC 주소를 학습한다.
  
## VLAN (Virtual LAN)
- : VLAN은 하나의 스위치를 여러 개의 논리적 네트워크로 나누는(분리하는) 기술
- **사용하는 이유**
  - 네트워크 분리
  - 보안 향상
  - 브로드캐스트 감
  
### VLAN 예시
- VLAN1 : A B C D
- VLAN2 : E F G H
  - 같은 스위치에 연결되어 있어도
  - -> 서로 다른 네트워크로 동작

### VLAN 특징
- VLAN마다 브로드캐스트 도메인이 분리
- 서로 다른 VLAN 간에는 직접 통신 불가

### VLAN 간 통신
- VLAN 간 통신을 하려면
- -> 라우터 또는 L3 스위치 필요

## 정리
### 브로드캐스트 도메인 vs 충돌 도메인

| 구분 | 설명 |
|---|---|
| 충돌 도메인 | 충돌이 발생할 수 있는 네트워크 범위 |
| 브로드캐스트 도메인 | 브로드캐스트 프레임이 전달되는 범위 |

- 허브 → 하나의 충돌 도메인  
- 스위치 → 포트별 충돌 도메인  
- VLAN → 브로드캐스트 도메인 분리
