# 3. 네트워크 계층 - IP
## IP의 목적
- IP(Internet Protocol)는 네트워크 계층에서 사용되는 핵심 프로토콜로 다음 두 가지 목적을 가진다.
### 주소 지정(Addressing)
- 네트워크에서 송신지와 수신지를 식별하기 위해 IP주소를 사용한다.
- IP 패킷 헤더에는 다음 정보가 포함된다.
  - 송신지 IP 주소
  - 수신지 IP 주소
- 이를 통해 네트워크 장비가 패킷을 목적지까지 전달할 수 있다.
### 단편화 (Fragmentation)
- 패킷의 크기가 네트워크에서 허용되는 최대 전송 크기(MTU)보다 클 경우
  - 패킷을 여러 개로 나누어 전송한다.
- 수신 측에서는 이를 다시 조립하여 원래 데이터로 복원한다.
## IP의 특징
1) 비연결형 통신 (Connectionless)
- IP는 데이터를 전송하기 전에 연결을 설정하지 않는다.
- 바로 패킷 전송 가능
=> 이러한 방식은 빠르지만 신뢰성을 보장하지 않는다. 
2) 신뢰할 수 없는 통신 (Unreliable)
- IP는 다음 기능을 제공하지 않는다.
  - 패킷 전달 보장
  - 패킷 순서 보장
  - 중복 패킷 처리
  - 오류 복구
- 이러한 기능은 전송 계층의 TCP가 담당한다.
## IPv4 주소
`32bit = 4byte`
- 8비트씩 4개로 구성된다.
> ex) `192.168.0.1`
>  - 각 숫자는 [0 ~ 255]범위를 가진다.
- 각 8비트 단위를 옥텟 (octet) 이라고 한다.
## IPv6
- : IPv4 주소 부족 문제를 해결하기 위해 등장한 프로토콜이다.
- NAT 의존도를 줄이기 위해 설계되었다.

| 구분  | IPv4  | IPv6 |
| ----- | ----- | ------ |
| 주소 길이 | 32bit | 128bit |
| 표현 방식 | 10진수  | 16진수   |
| 주소 개수 | 약 43억 | 사실상 무한 |
> ex) IPv6 주소
> - `2001:0230:abcd:ffff:0000:0000:ffff:1111`

## 라우터와 라우팅
- 서로 다른 네트워크 간 통신은 라우터(router) 를 통해 이루어진다.
- **라우터의 역할**
  - IP 주소 기반 패킷 전달
  - 최적 경로 선택
    - 이 과정을 라우팅 (Routing) 이라고 한다.
## MTU와 단편화
### MTU (Maximum Transmission Unit)
- : 네트워크에서 한 번에 전송할 수 있는 최대 데이터 크기
- 일반적인 이더넷 환경
  - MTU = 1500 byte (Payload 기준)
### IP 단편화 (Fragmentation)
- 패킷 크기가 MTU보다 클 경우
  - 패킷을 여러 개로 나누어 전송한다.
- 수신 측에서 이를 재조립한다.
### 단편화 관련 필드
- IP 헤더에는 다음 필드가 존재한다.

| 필드              | 설명                 |
| --------------- | ------------------ |
| Identification  | 같은 데이터에서 분할된 패킷 구분 |
| Flags           | 단편화 여부 표시          |
| Fragment Offset | 패킷 순서 정보           |

## 경로 MTU 발견(Path MTU Discovery)
- : 단편화를 줄이기 위해 사용하는 기술이다.
- 네트워크 경로에서 가장 작은 MTU를 찾아 해당 크기로 패킷을 전송한다.

## IP 주소 구조
- IP 주소는 두 부분으로 구성된다.
- IP 주소 = 네트워크 주소 + 호스트 주소
  - IP 주소의 네트워크 부분과 호스트 부분은 서브넷 마스크에 의해 결정된다.
> ex) IP : `172.16.12.45`
>  - Subnet : `255.255.0.0`
>  - 네트워크 : `172.16`
>  - 호스트 : `12.45`

| 구성      | 의미      |
| ------- | ------- |
| 네트워크 주소 | 네트워크 식별 |
| 호스트 주소  | 장치 식별   |

## 클래스풀 주소 체계
- 초기 IP 주소 관리 방식으로 네트워크 크기에 따라 클래스를 나누었다.
### A 클래스
- 범위 : `0.0.0.0 ~ 127.255.255.255`
- 네트워크 : 8bit
- 호스트 : 24bit
- **대규모 네트워크에 사용**
### B 클래스
- 범위 : `128.0.0.0 ~ 191.255.255.255`
- 네트워크 : 16bit
- 호스트 : 16bit
- **중형 네트워크**
### C 클래스
- 범위 : `192.0.0.0 ~ 223.255.255.255`
- 네트워크 : 24bit
- 호스트 : 8bit
- **소규모 네트워크**

## 네트워크 주소와 브로드캐스트 주소
- 모든 IP 주소를 호스트에 할당할 수 있는 것은 아니다.
> ex) `172.16.12.45`
> `10101100.00010000.00001100.00101101`
### 네트워크 주소
- 호스트 부분이 모두 0인 네트워크 자체를 의미한다.
> ex) `172.16.0.0`
> `10101100.00010000.00000000.00000000`
### 브로드캐스트 주소
- 호스트 부분이 모두 1인 네트워크 자체를 의미한다.
> ex) `172.16.255.255`
> `10101100.00010000.11111111.11111111`
## 예약 IP 주소
- 특정 목적을 위해 예약된 IP 주소가 존재한다.

| 범위                            | 용도     |
| ----------------------------- | ------ |
| `0.0.0.0`                       | 현재 호스트 |
| `127.0.0.0 ~ 127.255.255.255`   | 루프백    |
| `10.0.0.0 ~ 10.255.255.255`     | 사설 IP  |
| `172.16.0.0 ~ 172.31.255.255`   | 사설 IP  |
| `192.168.0.0 ~ 192.168.255.255` | 사설 IP  |

### 루프백 주소
- 대표적인 루프백 주소 - `127.0.0.1`
- 의미
  - localhost
  - 자기 자신을 가리키는 주소이다.

## 클래스리스 주소 체계 (Classless Addressing)
- 초기 IP 주소 체계는 A, B, C 클래스로 네트워크를 구분했다.
- 하지만 IP 주소 낭비 문제가 발생하였다.
- 이를 해결하기 위해 등장한 것이 클래스리스 주소 체계 (Classless Addressing)이다.
- **특징**
  - 네트워크와 호스트를 유연하게 구분
  - 서브넷 마스크(subnet mask) 사용
### 서브넷 마스크
- 서브넷 마스크는 IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 값이다.

| 구분 | 의미      |
| -- | ------- |
| 1  | 네트워크 부분 |
| 0  | 호스트 부분  |
> ex) 
>  - IP 주소      : `192.168.200.102`
>  - 서브넷 마스크 : `255.255.255.0` 
- 이를 AND 연산하면 네트워크 주소를 구할 수 있다.
  - 결과 : `192.168.200.0` 

### CIDR 표기법
- IP 주소와 서브넷 마스크를 함께 표현하는 방식이다.
- **표현 방식**
  - IP주소 / prefix
> ex) `192.168.20.3/30`
- 의미
  - 앞 30비트가 네트워크
  - 나머지 2비트가 호스트

## 공인 IP 주소와 사설 IP 주소
- IP 주소는 크게 두 종류로 나뉜다.

| 구분    | 설명                |
| ----- | ----------------- |
| 공인 IP | 인터넷에서 사용되는 고유 주소  |
| 사설 IP | 내부 네트워크에서 사용하는 주소 |

### 공인 IP 주소 (Public IP)
- 전 세계에서 유일한 IP 주소
- **특징**
  - 인터넷에서 사용
  - ISP가 할당
  - 외부 통신 가능
> ex) `163.152.3.171`
- 웹에서 확인 가능
  - what is my ip address?
### 사설 IP 주소 (Private IP)
- 내부 네트워크에서만 사용하는 IP 주소
- **특징**
  - 인터넷에서 직접 사용 불가
  - 공유기(NAT)를 통해 외부 통신
- **사설 IP 범위**

| 범위                            | CIDR           |
| ----------------------------- | -------------- |
| `10.0.0.0 ~ 10.255.255.255`     | `10.0.0.0/8`     |
| `172.16.0.0 ~ 172.31.255.255`   | `172.16.0.0/12`  |
| `192.168.0.0 ~ 192.168.255.255` | `192.168.0.0/16` |
  > - ex) `192.168.0.10`
  > - `10.0.0.5`

## IP 주소의 할당 방식
- 호스트에 IP 주소를 할당하는 방법은 두 가지가 있다.

| 방식    | 설명      |
| ----- | ------- |
| 정적 할당 | 수동 설정   |
| 동적 할당 | DHCP 사용 |

## 정적 IP 할당 (Static IP)
- : 관리자가 직접 IP 주소를 설정하는 방식
- 설정 시 필요한 정보
  - IP 주소
  - 서브넷 마스크
  - 게이트웨이
  - DNS 서버
### 기본 게이트웨이 (Default Gateway)
- 다른 네트워크로 나가기 위한 기본 경로
- 일반적으로 라우터(공유기) 주소이다.
> ex) `192.168.0.1`

## DNS (Domain Name System)
- : DNS는 도메인 이름을 IP 주소로 변환하는 시스템이다.
- **필요 이유**
  - 컴퓨터는 IP 주소로 통신하지만
  - 사람은 도메인 이름을 사용하기 때문이다.
- ex) google.com → 142.250.66.110
### DNS 동작 과정
- 1️⃣ 사용자가 도메인 입력
  - hanbit.com
- 2️⃣ DNS 서버에 질의
  - hanbit.com 의 IP 주소 요청
- 3️⃣ DNS 서버 응답
  - IP 주소 반환
- 4️⃣ 해당 서버와 통신

### 대표 DNS 서버

| 제공자        | DNS     |
| ---------- | ------- |
| Google     | 8.8.8.8 |
| Cloudflare | 1.1.1.1 |

## DHCP (Dynamic Host Configuration Protocol)
- : DHCP는 자동으로 IP 주소를 할당하는 프로토콜
- **특징**
  - 동적 IP 주소 할당
  - 자동 네트워크 설정
  - 대부분의 네트워크에서 사용
- 일반적으로 공유기(라우터) 가 DHCP 서버 역할을 한다.

### DHCP 특징
- IP 주소 임대(lease) 방식
- 일정 시간 후 반납
- 재할당 가능

## ICMP (Internet Control Message Protocol)
- : 네트워크 상태를 확인하거나 오류를 보고하는 제어 프로토콜이다.
- **역할**
  - 네트워크 오류 보고
  - 네트워크 상태 진단 

### ICMP 메시지 종류
- **오류 보고**

| 메시지                             | 의미         |
| ------------------------------- | ---------- |
| Destination network unreachable | 네트워크 도달 불가 |
| Destination host unreachable    | 호스트 도달 불가  |
| Destination port unreachable    | 포트 도달 불가   |
| TTL expired                     | TTL 만료     |

- **네트워크 진단**

| 메시지          | 의미      |
| ------------ | ------- |
| Echo request | ping 요청 |
| Echo reply   | ping 응답 |

### TTL (Time To Live)
- IP 패킷의 수명
- 라우터를 하나 지날 때마다 TTL이 1 감소
- TTL = 0 이 되면
  - 패킷 폐기
  - ICMP 메시지 반환
### Hop
- 패킷이 라우터를 한 번 통과하는 것
- 라우터 1개 = 1 hop
### 네트워크 진단 명령어
- **ping**
  -  네트워크 연결 상태 확인
  - ping google.com
- **traceroute**
  -  패킷이 지나가는 경로 확인
  -  traceroute google.com
  -  Windows
    - tracert google.com 
## ARP (Address Resolution Protocol)
- ARP는 IP 주소를 MAC 주소로 변환하는 프로토콜이다.
- ARP는 동일 네트워크(Local Network)에서 동작한다.
- 이유
  - 네트워크에서는 
    - IP 주소
    - MAC 주소
  - 두 가지 주소가 사용되기 때문이다. 
### ARP 동작 과정
- 1️⃣ 송신 호스트가 ARP 요청
  - `192.168.0.5` 의 MAC 주소 누구인가?
- 2️⃣ 동일 네트워크에 브로드캐스트
- 3️⃣ 해당 IP를 가진 호스트가 응답
  - 내 MAC 주소는 xx:xx:xx:xx:xx
- 4️⃣ ARP 테이블 저장

### ARP 요청 특징
- 브로드캐스트
- 네트워크 모든 호스트가 수신

### ARP 테이블
- IP 주소와 MAC 주소의 매핑 정보를 저장하는 테이블
- 확인 명령어 : arp -a

## IP와 MAC 주소 관계

| 구분     | 역할    |
| ------ | ----- |
| IP 주소  | 논리 주소 |
| MAC 주소 | 물리 주소 |

- 비유

| 개념     | 비유      |
| ------ | ------- |
| IP 주소  | 집 주소    |
| MAC 주소 | 실제 집 위치 |

### 네트워크 계층 핵심 정리
- **IP 특징**
  - 비연결형
  - 신뢰성 없음
  - Best-effort 전달
- **IP 주소 종류**

| 종류    | 설명       |
| ----- | -------- |
| 공인 IP | 인터넷에서 사용 |
| 사설 IP | 내부 네트워크  |

- **IP 할당 방식**

| 방식    | 설명    |
| ----- | ----- |
| 정적 할당 | 수동 설정 |
| 동적 할당 | DHCP  |

- **네트워크 진단**

| 명령어        | 기능    |
| ---------- | ----- |
| ping       | 연결 확인 |
| traceroute | 경로 확인 |

- **주소 변환**
  - IP → MAC => ARP
