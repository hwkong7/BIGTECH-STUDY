## 고가용성: 로드 밸런싱과 스케일링

### 가용성

- 주어진 특정 기능을 실제로 수행할 수 있는 시간의 비율

$$
가용성 = \frac{업타임}{업타임 + 다운타임}
$$

- 업타임 : 정상적인 사용 시간
- 다운타임 : 모종의 이유로 인해 정상적인 사용이 불가능한 시간

# 🔥 면접에서 나올 가능성 높은 추가 질문 & 보완 포인트

## 1️⃣ 프록시 vs 게이트웨이 (⭐ 자주 나옴)

### 핵심 차이

| 구분 | 프록시 (Forward Proxy) | 게이트웨이 (Reverse Proxy) |
| --- | --- | --- |
| 기준 | 클라이언트 입장 | 서버 입장 |
| 목적 | 클라이언트 보호 | 서버 보호 |
| 대표 예시 | 회사 인터넷 필터링 | Nginx, Cloudflare |
| 서버 인식 | 서버는 클라이언트 모름 | 클라이언트는 실제 서버 모름 |

👉 면접 한 줄 정리

- **Forward Proxy → 클라이언트 대신 요청**
- **Reverse Proxy → 서버 대신 응답**

---

## 2️⃣ 포워드 프록시 vs 리버스 프록시 (⭐ 필수)

### Forward Proxy

- 클라이언트 앞에 있음
- **클라이언트가 인터넷에 나갈 때 사용**
- 예: 회사, 학교에서 특정 사이트 차단

### Reverse Proxy

- 서버 앞에 있음
- **외부 요청을 서버 대신 받아줌**
- 예: Nginx, CDN, 로드 밸런서

👉 핵심 키워드

- Forward → "클라이언트 보호"
- Reverse → "서버 보호 + 성능 향상"

---

## 3️⃣ 로드 밸런싱 알고리즘 (조금만 더 추가하면 좋음)

### 추가하면 좋은 것

- **IP Hash**
    - 같은 사용자는 항상 같은 서버로
    - 로그인 유지에 좋음
- **Weighted Round Robin**
    - 서버 성능에 따라 가중치 다르게 분배

👉 면접 포인트

> "단순 라운드로빈은 서버 성능 차이를 반영 못함 → 가중치 필요"
> 

---

## 4️⃣ L4 vs L7 로드 밸런서 (⭐ 중요)

### L4 (Transport Layer)

- TCP/UDP 기준
- 속도 빠름
- 단순 분배

### L7 (Application Layer)

- HTTP 기준
- URL, 헤더 기반 분기 가능
- 예: `/api`는 A 서버, `/img`는 B 서버

👉 한 줄 정리

- **L4 = 빠름**
- **L7 = 똑똑함**
