# 로드 밸런싱과 스케일링

## 가용성

> 주어진 특정 기능을 실제로 수행할 수 있는 시간의 비율
> 
- 업타임
    - 정상적인 사용 시간
- 다운타임
    - 정상적인 사용이 불가능한 시간
    - 과도한 트래픽
    - 예기치 못한 소프트웨어 상의 오류 또는 하드웨어 장애
    - 보안 공격이나 자연재해

```sql
가용성 = 업타임 / 업타임 + 다운타임
```

- 99.999% 이상 → 안정적
- 원인을 모두 차단하기는 현실적으로 어려움 → 문제가 발생하더라도 계속 기능할 수 있도록 설계하는것이 목표

## 로드밸런싱

> 트래픽의 고른 분배를 위해 사용되는 기술
> 
- 로드 밸런서에 의해 수행
    - 클라이언트 - 서버 사이에 위치
    - 네트워크 장치 or 소프트웨어
    - Nginx
- 로드 밸런싱 알고리즘
    - 서버가 여러 개일 경우 어떤 서버에 요청을 전달해야할까
    - 라운드 로빈
    - 최소 연결 알고리즘
    - 해시 링
